Then wait. Once specs are received (loose or detailed):
1. Write a plan — listing what will be created, reused, or modified. No code yet.
2. Wait for validation (approved / adjusted / rejected).
3. Execute in small, coherent commits so changes can be followed in the editor and cherry-picked if needed.

Before asking to commit, update `site-plan.md` if pages were added, removed, or their purpose changed.
