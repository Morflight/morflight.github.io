# Intentional Drift

This file documents **deliberate** deviations from the Dev workspace standards (template, specs, standards). `/harmonize` aggregates these across all projects to identify improvements worth propagating.

Drifts are high-signal: if this project found a better way to do something, it should flow back to the template and forward to other projects. When a drift is propagated and becomes the new standard, remove its entry here.

## Format

Each entry is a heading with a scope tag and a short explanation of **why** the deviation exists.

```markdown
### [scope] Short description
Why: reason this project deviates intentionally
```

**Scope** is one of:
- `file` — a .claude/ file that differs from the template on purpose
- `structure` — a folder layout deviation from specs.md
- `ci` — a CI/CD convention deviation from standards.md
- `skill` — a skill that was modified or removed on purpose
